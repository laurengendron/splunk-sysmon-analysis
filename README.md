# Sysmon CSV Ingestion & Analysis in Splunk Cloud

## Project Overview
This project demonstrates how to ingest and analyze **Sysmon event logs** using **Splunk Cloud**. The dataset includes network connection events generated by Sysmon, a Windows system monitoring tool.

The goal is to:
- Ingest CSV log data into Splunk
- Assign a proper sourcetype and index
- Run SPL queries to extract meaningful insights
- Showcase troubleshooting and Splunk workflow skills

---

## Dataset
- Source: Sysmon CSV export
- Contents: Network connection events with fields such as `UtcTime`, `Image`, `ProcessId`, `SourceIp`, `DestinationIp`, `DestinationPort`, `User`, and more.

---

## Splunk Workflow

1. **Upload CSV to Splunk Cloud**
   - Navigate to **Add Data → Upload → CSV**
   - Assign a **custom sourcetype**: `sysmon_csv`
   - Select or create an index (e.g., `sysmon`)

2. **Verify Ingestion**
   - Search in the selected index:

   ```spl
   index=sysmon sourcetype=sysmon_csv
   | table UtcTime, Image, ProcessId, SourceIp, DestinationIp, DestinationPort, User
   
   source="Sysmon.csv"   index=main

   source="Sysmon.csv"
    | stats count by index, sourcetype
